# 仕様書

## 目的
Raspberry Pi 5に接続されたUSBカメラのライブ映像と、MLX90640の温度データを1画面でリアルタイム表示する。

## 対象ユーザー
- Raspberry Pi 5上でセンサーとカメラを監視したい開発者/運用者

## 主要機能
- USBカメラのライブ映像表示
- MLX90640温度データのヒートマップ表示
- MLX90640の最高温度/最低温度の表示

## 画面一覧
- 監視ダッシュボード（単一画面、スマホ画面想定）

## 画面要件
- 1画面にUSBカメラ映像と温度ヒートマップを並べて表示
- 最高/最低温度を目立つ位置に表示
- メインカラーはダークブルー

## データ/状態
- USBカメラ映像: ストリーミング（リアルタイム）
- MLX90640温度: リアルタイム更新（できれば60fps、厳しい場合は10fps刻みで下げる）
- 表示状態: 現在のヒートマップ、max/min温度

## ルーティング
- `/` : 監視ダッシュボード
- `/stream` : USBカメラ映像ストリーム（実装方式は後述）
- `/thermal` : MLX90640温度データ取得API（JSON）

## UI/デザイン方針
- スマホ画面を前提にした1カラム構成
- ダークブルーを基調とした配色
- 情報は「映像」「ヒートマップ」「max/min」の順に視線誘導

## 技術構成
- Backend: Flask
- Frontend: TypeScript + HTML/CSS
- センサー: MLX90640（I2C接続）
- 実行環境: Raspberry Pi 5

## 非機能要件
- リアルタイム表示（可能であれば60fps、厳しい場合は10fps刻みで下げる）
- 同一Raspberry Pi上で完結（カメラ/センサー同居）

## 受け入れ条件
- 1画面でUSBカメラのライブ映像が確認できる
- MLX90640のヒートマップがリアルタイムで更新される
- max/min温度が画面上に表示される
- スマホ画面で閲覧でき、ダークブルー基調のUIになっている

## スコープ/優先度
- 必須: ライブ映像、ヒートマップ、max/min表示
- 可能なら: 60fps相当の更新（厳しい場合は10fps刻みで下げる）
